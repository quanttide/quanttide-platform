# CONTRIBUTING

## 文件组织

每个 `.tf` 文件按上下分为两个区域：

**管理员（环境无关）**
- 声明 Vault 运行时的资源：secrets engine、policies
- 本地和云共用，上云不修改

**部署层（环境专属）**
- 生成本地配置文件、随机密码、启动脚本、备份文件
- 上云时整个区域替换为云模块

## 规则

- 文件内先用 `# ====` 分隔线标出管理层和部署层
- 管理层在上，部署层在下
- 部署层依赖管理层，不得反向引用
- `locals` 归入所属区域的尾部
- 涉及敏感值的备份文件使用 `count = var.backup_dir != null ? 1 : 0` 条件创建
