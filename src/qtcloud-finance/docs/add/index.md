# 架构设计文档

## 首页

量潮财务记账助手，基于 Beancount 复式记账格式，提供自然语言记账和银行流水对账功能。

## 领域建模

### 设计原则

| 原则 | 说明 |
|------|------|
| 本地优先 | 数据存储在本地文件，不依赖云端服务 |
| 隐私保护 | LLM 使用本地 Ollama，账本数据不外传 |
| 复式记账 | 遵循 Beancount 规范，确保账目平衡 |
| 人机协作 | 智能体提供建议，人工确认后执行 |

### 领域实体

| 对象 | 说明 |
|------|------|
| 银行流水 | 银行导出的交易记录 (CSV) |
| Beancount 账本 | 复式记账格式文件 |
| 调账记录 | 确认后的调账分录 |

### 领域服务

| 智能体 | 职责 | 输入 | 输出 |
|--------|------|------|------|
| **记账员** | 记录交易 | 自然语言 | Beancount 记账 |
| **对账员** | 核对差异 | 银行流水 + 账本 | 差异 / 调账建议 |

### 上下文

```
┌─────────────┐    记账    ┌─────────────┐
│  自然语言   │ ─────────► │   记账员    │
└─────────────┘            └──────┬──────┘
                                 │ Beancount
                                 ▼
┌─────────────┐    对账    ┌─────────────┐    差异/建议    ┌─────────────┐
│  银行流水   │ ─────────► │   对账员    │ ────────────► │  人工确认   │
└─────────────┘            └──────┬──────┘                └──────┬──────┘
                                 │ Beancount                      │ 调账记录
                                 ▼                               ▼
                          ┌─────────────┐              ┌─────────────┐
                          │  Beancount  │ ◄─────────── │   账本      │
                          │    账本    │              └─────────────┘
                          └─────────────┘
```

### 主要功能

| 功能 | 智能体 | 输入 | 输出 |
|------|--------|------|------|
| 自然语言记账 | 记账员 | 用户口语描述 | Beancount 分录 |
| 银行流水对账 | 对账员 | CSV 流水 + 账本 | 差异报告 + 调账建议 |

### 验收标准

| 场景 | 验收条件 |
|------|----------|
| 记账 | 输入"午餐80块"，输出格式正确的 Beancount 分录，借贷平衡 |
| 对账 | 银行流水与账本存在差异时，能识别并输出调账建议 |
| 验证 | 生成的账本可通过 `bean-check` 验证 |

## 技术实现

### 技术栈

- **Python** - 开发语言
- **Textual** - TUI 框架
- **Pydantic Settings** - 配置管理
- **Beancount** - 账本解析与验证
- **Ollama** - 本地 LLM

### 存储

- 本地文件
- 阿里云 OSS
- AWS S3

### 配置

环境变量：`ALIBABA_CLOUD_ACCESS_KEY_ID`、`OSS_ENDPOINT`、`OSS_BUCKET_NAME` 等

详见 [.env.example](../../.env.example)
