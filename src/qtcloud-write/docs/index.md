# 量潮写作云

## 产品定位

**集体写作平台**：为团队和个人提供协作式写作体验，支持文档管理、版本控制、协作编辑和内容发布。

## 核心功能

### 1. 文档管理
- 文档库：创建、分类、管理文档
- 文件夹结构：多层级目录组织
- 文档状态：草稿/审核中/已发布/已归档
- 标签分类：自定义标签，多维度分类

### 2. 协作编辑
- 实时协作：多人同时编辑同一文档
- 权限控制：查看/评论/编辑/管理员
- 编辑锁：防止编辑冲突
- 光标同步：显示协作者位置

### 3. 版本控制
- 自动保存：定时保存编辑内容
- 版本历史：查看和恢复历史版本
- 变更对比：可视化展示版本差异
- 分支写作：创建分支进行独立修改

### 4. 评论与反馈
- 行内评论：针对特定内容添加评论
- 评论讨论：在评论下进行回复讨论
- 评论状态：待处理/已解决/已关闭
- @提及：通知特定协作者

### 5. 内容发布
- 多格式导出：Markdown/HTML/PDF/Word
- 发布管理：定时发布、撤回发布
- 公开链接：生成只读分享链接
- 嵌入支持：嵌入到其他网站

## 用户角色

### 个人写作者
- 管理个人文档库
- 使用富文本编辑器写作
- 导出和发布内容
- 接收反馈和评论

### 团队协作者
- 参与团队文档协作
- 添加评论和建议
- 跟踪文档变更
- 协同完成写作任务

### 内容审核者
- 审核文档内容
- 批准或驳回发布请求
- 管理评论和反馈
- 维护内容质量

## 数据模型

### 文档 (Document)
```
- id: 唯一标识
- title: 标题
- content: 内容（Markdown）
- author_id: 作者ID
- folder_id: 所属文件夹
- status: 文档状态
- tags: 标签列表
- word_count: 字数统计
- created_at: 创建时间
- updated_at: 更新时间
- published_at: 发布时间
```

### 文件夹 (Folder)
```
- id: 唯一标识
- name: 文件夹名称
- parent_id: 父文件夹ID
- owner_id: 所有者ID
- created_at: 创建时间
```

### 版本 (Version)
```
- id: 唯一标识
- document_id: 文档ID
- content: 内容快照
- author_id: 修改者ID
- message: 版本说明
- created_at: 创建时间
```

### 评论 (Comment)
```
- id: 唯一标识
- document_id: 文档ID
- author_id: 评论者ID
- content: 评论内容
- selection_start: 选区开始
- selection_end: 选区结束
- parent_id: 父评论ID
- status: 评论状态
- created_at: 创建时间
```

### 协作成员 (Collaborator)
```
- id: 唯一标识
- document_id: 文档ID
- user_id: 用户ID
- role: 角色（viewer/commenter/editor/admin）
- invited_by: 邀请者ID
- created_at: 加入时间
```

## 技术架构

### 前端
- Web端：React/Vue 富文本编辑器
- 移动端：Flutter 跨平台应用
- CLI端：命令行工具

### 后端
- API服务：FastAPI
- 协作引擎：WebSocket 实时同步
- 数据库：PostgreSQL
- 缓存：Redis
- 存储：对象存储（图片/附件）

### 编辑器技术
- 富文本编辑：ProseMirror / TipTap
- Markdown编辑：CodeMirror / Monaco
- 冲突解决：CRDT / OT算法
- 实时同步：WebSocket + Operational Transform

## 使用场景

### 场景一：个人写作
```
用户创建新文档
-> 使用 Markdown 编辑器写作
-> 系统自动保存
-> 添加标签分类
-> 导出为 PDF 发布
```

### 场景二：团队协作
```
团队创建项目文档
-> 邀请成员加入协作
-> 分配写作任务
-> 实时协作编辑
-> 添加评论讨论
-> 审核通过后发布
```

### 场景三：内容审核
```
作者提交文档审核
-> 审核者收到通知
-> 添加行内评论和建议
-> 作者根据反馈修改
-> 审核通过，正式发布
```
