# 量潮阅读云

## 产品定位

**集体阅读平台**：为团队和个人提供协作式阅读体验，支持书籍管理、阅读进度跟踪、读书笔记分享和阅读讨论。

## 核心功能

### 1. 书籍管理
- 个人书架：添加、分类、管理个人书籍
- 团队书库：共享团队推荐阅读书目
- 阅读状态：想读/在读/已读/弃读
- 标签分类：自定义标签，多维度分类

### 2. 阅读进度
- 进度记录：章节/页数/百分比
- 阅读时长：自动或手动记录阅读时间
- 阅读计划：设定阅读目标和截止日期
- 进度可视化：图表展示阅读轨迹

### 3. 读书笔记
- 划线摘录：记录书中精彩段落
- 读书感想：记录阅读时的思考
- 章节笔记：按章节组织笔记
- 标签索引：为笔记添加标签便于检索

### 4. 集体阅读
- 共读活动：发起团队共读一本书
- 阅读小组：创建或加入兴趣小组
- 进度同步：查看队友阅读进度
- 讨论区：围绕书籍进行讨论

### 5. 阅读分享
- 笔记分享：公开或指定分享读书笔记
- 书评发布：撰写和发布书评
- 推荐系统：基于阅读历史推荐书籍
- 阅读圈子：发现志同道合的读者

## 用户角色

### 个人读者
- 管理个人阅读书架
- 记录阅读进度和笔记
- 参与共读活动和讨论
- 发现新书和阅读伙伴

### 团队管理者
- 创建团队书库和推荐阅读
- 发起和管理共读活动
- 查看团队阅读统计
- 促进团队阅读文化

### 内容贡献者
- 撰写高质量书评
- 分享阅读方法和技巧
- 推荐优质书籍
- 引导阅读讨论

## 数据模型

### 书籍 (Book)
```
- id: 唯一标识
- title: 书名
- author: 作者
- isbn: ISBN号
- cover: 封面图片
- description: 简介
- tags: 标签列表
- created_at: 添加时间
```

### 阅读记录 (Reading)
```
- id: 唯一标识
- user_id: 用户ID
- book_id: 书籍ID
- status: 阅读状态
- progress: 阅读进度
- start_date: 开始日期
- finish_date: 完成日期
- rating: 评分
```

### 笔记 (Note)
```
- id: 唯一标识
- user_id: 用户ID
- book_id: 书籍ID
- chapter: 章节
- content: 笔记内容
- quote: 原文摘录
- page: 页码
- tags: 标签
- is_public: 是否公开
- created_at: 创建时间
```

### 共读活动 (ReadingGroup)
```
- id: 唯一标识
- name: 活动名称
- book_id: 书籍ID
- creator_id: 创建者ID
- start_date: 开始日期
- end_date: 结束日期
- members: 成员列表
- description: 活动说明
```

## 技术架构

### 前端
- Web端：响应式网页应用
- 移动端：Flutter 跨平台应用
- CLI端：命令行工具

### 后端
- API服务：FastAPI
- 数据库：PostgreSQL
- 缓存：Redis
- 搜索：Elasticsearch

### 存储
- 对象存储：书籍封面、用户头像
- 文件系统：阅读笔记、书评

## 使用场景

### 场景一：个人阅读管理
```
用户添加一本书到书架
-> 设置阅读状态为"在读"
-> 每天记录阅读进度
-> 阅读时随手记录笔记
-> 读完后撰写书评
```

### 场景二：团队共读
```
团队发起"每月一书"活动
-> 成员加入共读
-> 每周分享阅读进度和心得
-> 在讨论区交流阅读体会
-> 活动结束后总结分享
```

### 场景三：阅读推荐
```
用户浏览推荐书单
-> 查看其他读者的书评和评分
-> 添加感兴趣的书籍到书架
-> 加入该书的共读小组
```
