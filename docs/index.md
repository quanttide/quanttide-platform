# 量潮工作平台

## 叙事蓝图

量潮工作平台的产品不把功能裸露给用户，而是把用户放在一个情境中，让他们的每个选择都有可感知的结果。这一理念来自叙事工程（Narrative Engineering）。

### 定义

叙事蓝图是叙事工程的设计产物——它为每个产品定义了一套引导式体验框架：

叙事蓝图把复杂过程拆成用户可理解的 Scene（场景），每一步展示当前信息并让用户选择或确认，选择影响分支路径，最终完成从模糊到确认的转化。

这不是 UI 框架，也不是交互组件库——它是平台所有产品共用的一套认知路径设计范式。


### 叙事蓝图的优势

相比传统的"功能罗列型"或"看板管理型"产品，叙事蓝图驱动的产品有几个不同：

- 降低认知门槛：用户不需要先学会工具再做事，而是在做事的过程中自然学会工具
- 减少沟通损耗：每一步的分支和确认节点都是需求对齐点，防止"做完了才发现不是客户要的"
- 可复现、可教学：用户走过的每条叙事路径都可以回放、分享，转化为培训素材

### 平台产品中的体现

量潮课堂面向学员，解决的是不懂某个知识点的问题，通过教学视频和分支练习来授课，以课后测验作为确认节点，最终让学员获得知识。

量潮数据面向客户，解决的是说不清需求、看不懂处理过程的问题，通过数据预览和处理步骤可视化来展示，以需求确认、中间结果确认和验收确认作为确认节点，最终让客户获得数据产物。
