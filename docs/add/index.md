# 系统架构设计风格

## 领域驱动分层

系统以领域驱动设计为核心组织方式，分为用户界面层、应用层、领域层、基础设施层。每层有明确的职责边界，层间通过接口通信。

## 微服务化

每个领域产品独立部署、独立演进，通过聚合仓库组合为完整产品。从单体向微服务持续演进。

## 子模块即产品

每个 `apps/qtcloud-*` 是独立产品级子模块，拥有自己的生命周期、文档体系和部署流程。主仓库只负责编排，不持有领域逻辑。

## 事件驱动协作

跨领域业务流程通过事件驱动而非中心调度完成。每个域发布标准化领域事件，订阅相关域的变更信号。域之间通过事件契约（AsyncAPI 定义）耦合，不产生运行时依赖。事件格式遵循 CloudEvents 规范。详见 [领域事件传递机制](intro/domain_events.md)。
