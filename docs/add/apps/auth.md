# 身份云

## 一句话定义

身份云是量潮系统的身份底座——它是所有领域产品的共同前置依赖，提供组织、用户、权限的统一建模与生命周期管理。

## 定位

身份云不属于任何特定业务域，而是被所有域依赖的基础领域服务。它既是领域层产品（定义身份语义模型），又是基础设施（提供认证机制）。

## 领域语言

| 概念 | 说明 |
|------|------|
| Organization（组织） | 一个独立的租户，拥有自己的成员、配置和资源空间 |
| Workspace（工作空间） | 组织内的协作单元，隔离项目和数据 |
| User（用户） | 跨组织的自然人身份，可加入多个组织 |
| Membership（成员关系） | 用户与组织/工作空间的关联，包含角色分配 |
| Role（角色） | 权限集合的命名抽象，支持细粒度权限定义 |

## 领域事件

身份云通过领域事件与其他产品协作，不产生运行时依赖：

| 事件 | 触发时机 | 消费者示例 |
|------|----------|-----------|
| `org.created` | 新组织注册 | 资产云初始化默认资源，计费云创建设账 |
| `org.config.updated` | 组织设置变更 | 所有域同步配置 |
| `user.joined` | 用户加入组织 | 沟通云创建默认频道，知识云分配语义空间 |
| `user.role.updated` | 角色变更 | 各域重新计算权限缓存 |
| `user.left` | 用户离开组织 | 各域清理数据访问权限 |

## 架构

身份云分为两层：

```
┌─────────────────────────────────────┐
│  qtcloud-auth 领域层                │
│  领域语言 · 领域事件 · API 契约     │  ← 稳定，对消费者透明
├─────────────────────────────────────┤
│  Adapter（实现适配器）              │
│  Stack Auth │ SuperTokens │ Authlib │  ← 可替换，按阶段演进
└─────────────────────────────────────┘
```

领域层定义稳定的接口和事件契约，底层通过 Adapter 模式对接具体实现。消费者只依赖领域层，不受底层实现变更影响。

## 路线

身份云底层实现按阶段演进，每个阶段以不同的核心交换为代价：

| 阶段 | 实现 | 核心交换 | 风险 |
|------|------|---------|------|
| Phase 1 | Stack Auth | 速度 → 产品成熟度 | 未来可能成为瓶颈 |
| Phase 2 | SuperTokens | 稳定性 → 迁移成本 | 数据/用户迁移有损 |
| Phase 3 | Authlib（自研） | 完全控制 → 工程投入 | 长期维护成本最高 |

### Phase 1：Stack Auth — 速度优先

部署 Stack Auth 作为自托管身份服务，先跑通 OAuth 登录、用户管理、角色权限。此阶段的交换是"用现成产品换开发速度"——上线最快，但产品本身较新，远期可能成为瓶颈。Adapter 实现最小可行，只覆盖基本认证流程。

### Phase 2：SuperTokens — 稳定性优先

当系统规模增长、稳定性要求超过上线速度时，将 Adapter 切换到 SuperTokens。此阶段的交换是"用成熟系统换工程稳定性"——迁移过程涉及用户数据转换，有损不可避免，但迁移后获得的稳定性收益大于损失。领域层 API 和事件不受影响，消费者无感。

### Phase 3：Authlib — 完全控制

当系统达到需要深度定制的规模（自定义认证流程、特殊权限模型、合规审计），将 Adapter 切换到 Authlib 自研实现。此阶段的交换是"用工程复杂度换完全控制权"——代价最高，适合在身份本身成为产品核心竞争力的阶段入场。

## 迁移原则

1. **领域层不变**：任何时候不因底层实现变更改动领域语言和事件契约
2. **Adapter 隔离**：每个实现有独立 adapter 模块，切换不产生全局改动
3. **消费者无感**：所有领域产品不感知当前跑的是哪个底层实现

## 与其他产品的协同

身份云是唯一被所有领域产品依赖的服务。每个产品在消费身份云事件时，只关心自己需要的子集：
- 资产云：监听 `org.created` 初始化资源配额
- 议事云/执行云：监听 `user.role.updated` 调整参与者权限
- 知识云：监听 `user.joined` 分配语义空间
