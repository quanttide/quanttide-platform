# QtCloud - 跨域协作目录

## 定位

跨域协作目录是 QtCloud 的产品机制，对应 BRD 场景四"跨领域业务流程的平台断裂"。

它不是集成平台（不参与运行时），而是各领域平台声明协作契约、发现彼此能力、编排跨域流程的设计时和治理工具。

## 用户角色

- **域平台团队**：为所在领域平台声明发布/订阅的事件契约
- **业务流程设计者**：跨域编排业务流转，定义事件响应链

## 核心功能

### 事件目录

每个域平台在此声明自己发布和订阅的事件。其他域可浏览、搜索、查看事件 schema 和示例负载。

```
销售云 → 发布: deal.created, deal.closed, deal.updated
销售云 → 订阅: customer.verified（来自客户数据域）
商务云 → 发布: contract.signed, contract.invoiced
商务云 → 订阅: deal.closed（来自销售域）
```

### 契约版本管理

事件 schema 版本化，变更时自动通知所有订阅方。订阅方需确认接收变更或提出异议。

### 跨域流程模板

预定义的跨域业务流程，可直接启用：

| 模板 | 事件链 | 涉及域 |
|------|--------|--------|
| 成交转化 | deal.closed → 创建合同 | 销售云 → 商务云 |
| 订单交付 | contract.signed → 准备交付 | 商务云 → 资产云 |
| 开票收款 | contract.invoiced → 创建应收 | 商务云 → 财务云 |

### 兼容性检测

订阅方定义 Consumer-Driven Contract（期望的事件格式和约束），提供方发布契约变更时自动运行兼容性检测，不破坏任何消费者才允许上线。

## 用户故事

### 域平台注册事件

> 作为销售云团队，我需要在协作目录中声明 `deal.closed` 事件的结构和含义，让其他域了解我发布的内容，并在 schema 变更时通知所有订阅方。

### 发现并订阅事件

> 作为商务云团队，我可以在协作目录中浏览所有域发布的事件，找到 `deal.closed` 事件并订阅。目录自动验证我订阅的事件格式是否兼容。

### 编排跨域流程

> 作为业务流程设计者，我可以在协作目录中选择"成交转化"模板，配置销售云和商务云之间的事件映射，一键启用跨域流程。

### 契约变更通知

> 作为销售云团队，我需要修改 `deal.closed` 事件的 schema。协作目录自动检测到商务云的订阅依赖，通知我变更可能产生的影响，并要求我逐一确认或协调。

## 非目标

- 不实现事件传输（由事件总线负责）
- 不存储业务数据（事件负载指向各域数据）
- 不替代各领域平台的产品能力（只做衔接，不做垂直功能）
